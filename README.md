# 커리큘럼

## 1주차

**\[기술 학습\]**

첫째 주에는 서버의 기본 개념을 이해하고, 데이터베이스의 기초부터 접근 방법까지 심층적으로 학습합니다. 특히, 서버 프레임워크가 무엇인지 그 역할과 중요성을 파악하고, 서버 운영에 필수적인 데이터베이스의 종류와 특징을 탐구하며 가장 밀접한 관계를 가진 데이터베이스는 무엇인지 함께 고민해봅니다.

**\[실전 학습\]**

효과적인 이력서 작성은 성공적인 커리어의 첫걸음입니다. '좋은 이력서'의 정의가 무엇인지 함께 논의하고, 각자의 경험과 강점을 담은 이력서 초안을 작성해봅니다. 이후에는 서로 피드백을 주고받으며 이력서를 더욱 완성도 높게 다듬는 시간을 가질 예정입니다.

- 서버 개념
- 데이터베이스와 JPA

## 2주차

**\[기술 학습\]**

이번 주에는 API (Application Programming Interface) 설계에 집중합니다. 사용자의 요청을 처리하는 서버에서 API 정의의 중요성을 이해하고, API를 설계하는 핵심 구성 요소인 Controller의 역할에 대해 심층적으로 알아봅니다. 더불어, 좋은 API를 정의하기 위한 원칙과 전략을 탐구하며 견고한 서버 설계를 위한 기반을 다집니다.

기술 학습 후에는 직접 구현한 간단한 웹 애플리케이션을 실제 서버에 배포해보는 실습을 진행합니다. 이 과정에서 CI/CD(지속적 통합/지속적 배포) 파이프라인을 구축하며 개발, 테스트, 배포 과정을 자동화하는 방법을 익혀 소프트웨어 개발 효율성을 높이는 경험을 하게 됩니다.

**\[실전 학습\]**

성공적인 커리어를 위한 또 다른 핵심 요소인 포트폴리오 작성에 대해 논의합니다. '어떤 것이 좋은 포트폴리오인지', '나의 강점을 어떻게 보여줄 수 있을지' 함께 고민하며 포트폴리오 초안을 작성합니다. 작성된 초안을 바탕으로 서로 피드백을 주고받으며 각자의 포트폴리오를 더욱 매력적이고 효과적으로 다듬어 나가는 시간을 가집니다.

- API와 Controller&Service 계층 및 구현
- CI/CD와 깃허브 액션

## 3주차

**\[기술 학습\]**

이번 주에는 테스트의 중요성을 이해하고, 테스트 코드 작성의 필요성과 방법에 대해 심층적으로 학습합니다. 안정적인 서비스 운영을 위한 필수 요소인 테스트 코드를 효과적으로 작성하는 노하우를 익힙니다.

이어서 서버 개발의 핵심 기능 중 하나인 로그인 구현에 대해 다룹니다. Spring 개발자에게 필수적인 Spring Security 프레임워크를 함께 학습하며, 안전하고 견고한 로그인 시스템을 구축하는 방법을 탐구합니다.

**\[실전 학습\]**

성장을 위한 중요한 성찰의 과정인 회고 작성에 대해 알아봅니다. '이번 멘토링에서 무엇을 배우고 느꼈는지'를 중심으로, 좋은 회고를 작성하는 방법을 학습하고 직접 회고록을 작성해봅니다. 이후에는 서로의 회고를 공유하고 피드백하며, 더욱 효과적인 자기 성찰과 성장을 위한 시간을 가집니다.

- 테스트 코드와 테스트 주도 개발
- 스프링 시큐리티와 로그인

## 4주차

4주간 진행된 멘토링의 마무리 시간입니다. 그동안 쌓아온 지식과 경험을 바탕으로 만들어진 결과물을 발표하고, 서로 피드백을 주고받는 시간을 가집니다.

4주간의 과정을 통해 얼마나 성장했는지 점검하고, 앞으로 나아갈 방향을 함께 모색합니다.

- 결과물 발표
